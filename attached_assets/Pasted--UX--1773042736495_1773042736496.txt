너는 시니어 풀스택 개발자이자 UX 디자이너다.
나는 코딩을 전혀 모르는 초보자다.
그래서 복잡한 설명보다, 바로 실행 가능한 구조로 앱을 만들어줘.
오류가 나지 않도록 안정적으로 구현하고, 초보자도 수정하기 쉽게 코드를 작성해줘.

[프로젝트 개요]
앱 이름: Face Score

앱 한줄 설명:
AI가 매일 당신의 얼굴 표정(인상)을 분석해 점수로 보여주고,
더 좋은 인상을 만드는 방법까지 알려드립니다.

핵심 메시지:
“당신의 인상은 몇 점?”

앱 목적:
사용자가 셀카를 찍거나 사진을 업로드하면
얼굴 인상을 분석한 뒤 점수와 피드백을 제공하고,
매일 기록을 쌓아 더 좋은 인상을 만드는 습관을 돕는 앱을 만든다.

[중요]
나는 왕초보다.
그러니 아래 원칙을 꼭 지켜줘.

1. 처음부터 너무 복잡한 기능을 넣지 말고 MVP로 만들어라.
2. 한 번에 실행 가능한 상태로 만들어라.
3. 코드 파일 구조를 단순하게 유지하라.
4. 설치/실행 방법도 아주 쉽게 설명하라.
5. 에러가 나면 자동으로 방어 처리하라.
6. 비어 있는 상태, 로딩 상태, 실패 상태도 UI에 표시하라.
7. 모바일 화면 기준으로 예쁘고 직관적으로 만들어라.
8. 디자인은 깔끔한 뷰티테크/헬스테크 느낌으로 만들어라.

[기술 요구사항]
- React 기반 웹앱으로 만들어라.
- TypeScript 사용
- 모바일 우선 반응형 UI
- 초보자에게 익숙하도록 가능한 단순한 구조
- Tailwind CSS 사용
- 차트는 간단한 라이브러리 사용 가능
- 로컬 저장(localStorage) 사용 가능
- 첫 버전은 로그인 없이 동작
- 첫 버전은 실제 서버 DB 없이 프론트엔드 중심으로 구현
- 사진은 기본적으로 서버에 저장하지 않는다
- 추후 AI 분석 API를 연결할 수 있도록 구조를 나눠라

[앱 기능 범위: MVP]
이 앱은 아래 4개 화면이 필요하다.

1. 홈 화면
- 앱 이름 Face Score 표시
- 메인 카피: “당신의 인상은 몇 점?”
- 서브 설명:
  “AI가 매일 당신의 얼굴 표정(인상)을 분석해 점수로 보여주고,
   더 좋은 인상을 만드는 방법까지 알려드립니다.”
- 사진 업로드 버튼
- 카메라 촬영 버튼(가능하면)
- 최근 점수 카드 1개 표시
- 개인정보 보호 안내 짧게 표시:
  “사진은 기본적으로 저장되지 않습니다.”

2. 분석 결과 화면
- 업로드한 사진 미리보기
- 오늘의 인상 점수 0~100점
- 세부 분석 항목 3개:
  1) 친근함
  2) 생기
  3) 자신감
- 한줄 총평
- 더 좋은 인상을 위한 팁 3개
- 다시 촬영/다시 업로드 버튼
- 저장 버튼
- 홈으로 가기 버튼

3. 기록 화면
- 최근 7일 기록 리스트
- 날짜별 점수 표시
- 최근 평균 점수 표시
- 점수 변화 그래프 표시
- 기록이 없을 때 빈 상태 메시지 표시

4. 설정 화면
- 앱 소개
- 개인정보 안내
- “사진 저장 안 함” 기본 정책 설명
- 알림 기능은 아직 실제 동작하지 않아도 되지만 스위치 UI는 넣어도 좋다
- 향후 프리미엄 기능이 들어갈 자리를 간단히 비워둘 것

[디자인 요구사항]
디자인은 신뢰감 있고 부드럽고 세련된 느낌으로 해줘.
- 전체 배경은 밝고 깨끗하게
- 포인트 컬러는 블루/퍼플 계열
- 카드 UI 적극 사용
- 둥근 모서리
- 그림자 부드럽게
- 숫자 점수는 크게 강조
- 사용자가 “검사받는다”기보다 “코칭받는다”는 느낌이 들게 해라
- 지나치게 병원 느낌이나 무서운 진단 느낌은 피하라

[분석 로직]
첫 버전에서는 실제 고급 AI 모델이 없어도 된다.
우선은 “동작하는 프로토타입”을 만들어라.

그래서 아래 방식으로 구현해라:
- 사용자가 사진을 업로드하면 분석 중 로딩 UI를 1~2초 보여준다.
- 이후 임시 점수 분석 결과를 생성한다.
- 점수는 랜덤만 쓰지 말고, 일관성 있어 보이도록 만들어라.
- 예를 들어 친근함/생기/자신감 항목 점수를 조합해 총점을 계산하라.
- 결과 문구도 점수 구간별로 다르게 보이게 하라.

예시 규칙:
- 친근함, 생기, 자신감 각각 50~95 범위에서 생성
- 총점은 세 항목 평균값
- 총점이 높을수록 긍정적 코멘트
- 총점이 낮더라도 상처 주는 표현은 금지
- 항상 개선 가능성과 실천 팁 중심으로 작성

[피드백 문구 톤]
절대 외모 비하처럼 느껴지면 안 된다.
톤앤매너는 아래처럼 해라:
- 따뜻함
- 코칭형
- 자기관리형
- 긍정적 변화 가능성 강조
- 부담 없는 표현
- 매일 다시 도전하고 싶게 만드는 말투

예시 총평 스타일:
- “오늘은 차분하고 신뢰감 있는 인상이 돋보여요.”
- “조금만 표정을 부드럽게 하면 더 친근한 인상을 줄 수 있어요.”
- “전체적으로 안정적인 인상이지만, 생기를 더하면 훨씬 좋아질 수 있어요.”

예시 팁 스타일:
- “입꼬리를 살짝 올리면 더 부드러운 인상을 줄 수 있어요.”
- “정면 조명을 활용하면 얼굴 인상이 더 또렷하게 보일 수 있어요.”
- “눈가 긴장을 풀고 편안한 표정을 유지해보세요.”

[개인정보/윤리 요구사항]
이 앱은 얼굴 사진을 다루므로 아래를 지켜라.
- 기본적으로 서버 저장 없음
- 로컬에서만 일시적으로 다루는 구조 우선
- 사용자가 저장 버튼을 누를 때만 점수 기록 저장
- 사진 자체는 저장하지 말고 점수 데이터만 저장 가능
- 화면 어디엔가 “참고용 분석 결과”임을 표시
- 의학적 진단처럼 보이게 하지 말 것
- 성형/비하/차별적 표현 금지
- 감정적으로 상처 줄 표현 금지

[상태 관리]
복잡한 전역 상태 라이브러리 없이 간단하게 구현해라.
- 업로드 사진 미리보기 상태
- 분석 결과 상태
- 저장된 기록 상태
- localStorage 활용 가능

[저장 데이터 구조]
기록은 아래 구조 예시로 저장해라.

{
  id: string,
  date: string,
  totalScore: number,
  friendliness: number,
  vitality: number,
  confidence: number,
  summary: string,
  tips: string[]
}

[차트]
기록 화면에는 최근 7일 점수를 보여주는 간단한 선 그래프를 넣어라.
기록이 적어도 에러 없이 작동해야 한다.

[초기 더미 데이터]
첫 실행 시 기록이 비어 있으면
예시 기록 2~3개를 넣어도 되고, 완전히 비워도 된다.
둘 중 하나를 선택하되 사용자 경험이 자연스럽게 해라.

[카메라]
브라우저에서 카메라 접근이 가능한 환경이면 촬영 기능을 지원하고,
불가능하면 자동으로 파일 업로드 방식으로 대체하라.
오류 메시지도 친절하게 보여줘라.

[결과 생성 함수]
향후 실제 AI API를 연결하기 쉽게
분석 로직을 별도 함수 파일로 분리해라.
예:
- analyzeFaceMock()
- generateSummary()
- generateTips()

나중에 내가 API를 연결할 수 있도록
주석으로 “여기에 실제 AI API 연결 가능”이라고 표시해라.
